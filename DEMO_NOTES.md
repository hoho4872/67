# 한끼 밥상 탄소 계산기 - 개발 노트 & 데모 시나리오

> 프로젝트 시연 및 설명을 위한 참고 문서입니다.

---

## 파트 1. 프로젝트 소개 (~1분)

`[화면: 웹사이트 메인 화면 — 로그인 모달이 떠 있는 상태]`

안녕하세요, 저는 **"한끼 밥상 탄소 계산기"**라는 웹 서비스를 만들었습니다.

저희가 매일 밥을 먹잖아요. 근데 그 한 끼가 탄소를 얼마나 배출하는지는 잘 모르거든요.
예를 들어 삼겹살 한 인분이면 CO₂가 거의 5kg 가까이 나오는데, 이게 차로 20km 달린 거랑 비슷합니다.
이런 걸 눈에 보이게 보여주는 서비스를 만들고 싶었습니다.

그래서 구현한 기능을 크게 보면,
**회원 인증** — 가입, 로그인, 비번 찾기, 탈퇴까지 전부 되고요,
**탄소 계산** — 음식을 고르면 실시간으로 CO₂가 얼마인지 계산해주고,
**클라우드 저장** — 내 식단을 Supabase DB에 저장할 수 있고,
**모바일 반응형** — 핸드폰으로 봐도 안 깨지게 처리했습니다.

기술 스택은 프론트는 **HTML, CSS, JavaScript** 순수 바닐라로 만들었고, 백엔드는 **Supabase**를 썼습니다. Supabase가 인증이랑 DB를 다 제공해줘서 서버 없이도 풀스택처럼 구현이 가능했습니다.

## 파트 2. 프롬프트 전략 (~1분 20초)

`[화면: carbon_calculator_guide.md 파일 또는 AGENTS.md 파일을 에디터에서 보여주기]`

이번 프로젝트는 AI 코딩 에이전트를 활용해서 만들었는데요, 여기서 제일 중요한 게 **프롬프트를 어떻게 짜느냐**였습니다.

저는 안드레이 카파시라는 분의 철학을 참고했는데요, 이 분이 **"프롬프트는 곧 프로그래밍이다"**라고 했거든요. AI한테 "대충 만들어줘" 하면 결과도 대충 나옵니다. 그래서 맥락이랑 지시를 명확하게 줘야 합니다.

그래서 제가 쓴 방법이 **프롬프트 블루프린터**인데요, 쉽게 말하면 **설계도를 먼저 만들고 나서 코딩을 시키는 겁니다.**

`[화면: carbon_calculator_guide.md의 로드맵 부분 스크롤해서 보여주기]`

이 `carbon_calculator_guide.md`라는 파일이 그 설계도입니다. 여기에 DB 테이블 구조, 인증 설정 방법, 개발 5단계 로드맵, 테스트 체크리스트까지 다 적어놨거든요. AI가 이걸 보고 코드를 짜니까 결과물이 훨씬 일관되게 나옵니다.

`[화면: AGENTS.md 파일 보여주기]`

그리고 이 `AGENTS.md`는 AI한테 **"너 이렇게 행동해"**라고 규칙을 정해준 파일입니다. "코딩 전에 먼저 생각해라", "최소한의 코드만 써라", "요청한 부분만 고쳐라" 이런 규칙을 미리 세팅해놓으면 AI가 쓸데없는 코드를 막 추가하거나 엉뚱한 데를 건드리는 걸 막을 수 있습니다.

## 파트 3. 기능 시연 (~1분 40초)

자, 이제 실제로 돌아가는 걸 보여드리겠습니다.

### ① 로그인 가드

`[화면: 로그인 모달이 뜬 초기 화면]`

사이트에 들어오면 이렇게 로그인 창이 먼저 뜹니다. 여기서 닫기 버튼이 없어요. 비로그인 상태에서는 무조건 로그인을 해야 서비스를 쓸 수 있게 만들었습니다.

`[화면: 모달 바깥 클릭 → Shake 효과 시연]`

이렇게 닫으려고 하면 창이 부르르 떨립니다.

### ② 회원가입

`[화면: 회원가입 탭 클릭 → 폼 입력]`

회원가입 탭을 누르면 이메일, 비밀번호, 비밀번호 확인, 그리고 비밀번호 분실 시 쓸 힌트 질문이랑 답변을 입력합니다. 비밀번호는 영문+숫자 8자 이상이어야 하고, 안 맞으면 빨간 에러 메시지가 뜹니다.

`[화면: 가입 완료 → 자동 로그인 → 메인 화면 전환]`

가입하면 자동으로 로그인되고, 상단에 이메일이 표시됩니다.

### ③ 음식 선택 & 탄소 계산

`[화면: 음식 카드 클릭 시연 — 쌀밥, 김치찌개, 삼겹살 순서로 클릭]`

이제 음식을 골라보겠습니다. 쌀밥, 김치찌개, 삼겹살을 클릭하면 위에 나무 식탁에 이모지가 올라가고요, 아래쪽에 총 CO₂ 배출량이 실시간으로 계산됩니다. 승용차로 몇 km를 달린 건지, 소나무가 몇 그루 필요한지도 같이 보여줍니다.

`[화면: 기간 드롭다운을 1년으로 변경]`

여기서 기간을 1년으로 바꿔보면, 이 식단을 매일 먹었을 때 1년간 누적 배출량도 환산해서 보여줍니다.

### ④ 클라우드 저장

`[화면: "☁️ Supabase에 밥상 저장" 버튼 클릭]`

이 "Supabase에 밥상 저장" 버튼을 누르면 식사 이름을 입력하고, 실제로 클라우드 DB에 저장이 됩니다.

### ⑤ 비밀번호 찾기

`[화면: 로그아웃 → 로그인 모달 → "비밀번호 찾기" 클릭]`

비밀번호 찾기 기능도 있는데요, 이메일을 입력하면 가입 때 설정한 힌트 질문이 나오고, 답을 맞추면 새 비밀번호로 바꿀 수 있습니다. 이메일 인증 메일 없이 바로 복구가 되는 방식입니다.

### ⑥ 모바일 반응형

`[화면: F12 개발자 도구 → 디바이스 모드 375px로 전환]`

마지막으로 모바일 반응형인데요, 개발자 도구에서 375px로 줄여보면 카테고리 탭이 가로 스크롤로 바뀌고, 음식 카드가 3열로 정렬되고, 버튼도 세로로 배치됩니다. 모달 창도 모바일에 맞게 패딩이 조절됩니다.

## 파트 4. 느낀점 (~50초)

`[화면: 아무거나 또는 메인 화면 유지]`

이번 프로젝트 하면서 느낀 점 세 가지 말씀드리겠습니다.

**첫째, AI한테 시키려면 설계부터 해야 한다는 겁니다.** 그냥 "만들어줘" 하면 엉망으로 나오는데, 가이드 문서를 먼저 만들고 단계별로 시키니까 결과물이 확실히 달랐습니다.

**둘째, AI가 만든 코드도 꼭 검증해야 합니다.** 모바일 반응형 작업할 때 AI가 코드를 짜줬는데, 실제로 375px에서 열어보니까 가로 스크롤이 생기더라고요. 기존 CSS에 min-width 고정값이 걸려 있었는데 AI가 그걸 놓친 거였습니다. 직접 확인하고 잡아냈습니다.

**셋째, Supabase 같은 서비스를 쓰면 백엔드를 안 만들어도 된다는 게 진짜 편합니다.** 인증, DB, 보안 정책까지 SQL 몇 줄로 끝나니까 프론트에 집중할 수 있었습니다.

결론적으로, **좋은 설계 문서랑 좋은 프롬프트가 있으면 AI랑 같이 혼자서도 풀스택 웹을 만들 수 있다**는 걸 직접 느꼈습니다.

감사합니다.
